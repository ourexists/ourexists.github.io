### 一、行业背景与问题定义

在工业数字化与工业互联网持续深化的背景下，中小型及规模化工业项目对设备全生命周期管理（EAM）、运行状态监控（O&M）、过程数据采集（DAQ）以及多终端协同管理的需求日益增强。微信小程序、移动 APP 与 Web 管理平台已逐步成为工业信息化的标准交互形态。

当前市场主流解决方案多由设备厂商主导构建，其设备云平台在商业模式与技术架构上大致可分为以下几类：

#### 1. 厂商云平台的典型模式

* **平台免费 + 设备溢价模式**：通过提高设备单价覆盖平台运营成本
* **设备原价 + 平台阶梯收费模式**：按设备接入规模收费，免费额度通常难以覆盖真实项目规模

#### 2. 现有技术路线分析

##### （1）基于本地 SCADA 的设备管理

> 优势：
> * 组态画面高度可定制，贴合工艺流程；
> * 数据本地存储，信息安全性高。

> 局限：
> * 开发与维护成本高；
> * 跨厂商设备接入能力弱；
> * 组态渲染性能瓶颈明显，难以支撑大规模设备场景。

适用范围：小型项目（< 100 台设备）

##### （2）基于云端 SCADA 的设备管理平台

* **特点**：在本地 SCADA 基础上引入云端集中管理
* **风险**：核心生产数据需上云，存在数据合规与安全隐患

##### （3）产品化云端设备管理平台（SaaS）

> 优势：
> * 快速部署，零开发成本；
> * 标准化接入流程。

> 局限：
> * 功能与界面高度固化；
> * 不支持复杂业务定制与多厂商深度融合。

##### （4）产品化本地部署平台（MES / 工业平台）

> 优势：
> * 支持本地化部署，数据完全可控；
> * 高度定制化能力，支持跨厂商设备整合。

> 局限：
> * 初期实施与服务器运维成本较高；
> * 项目规模差异导致投入成本波动较大。

### 二、OMES 系统定位与设计目标

OMES 是一款产品化 + 本地化部署的工业设备运营管理平台，旨在产品稳定性、系统灵活性与实施成本之间取得最佳平衡，填补中小型工业项目在信息化投入与产出之间的空白。

系统设计围绕以下核心问题展开：

1. 如何实现多厂商、多协议设备的统一接入与管理
2. 如何在保留 SCADA 工艺组态优势的同时，避免其系统耦合与性能瓶颈
3. 如何提供 Web / 移动端 / 小程序 多终端一致的业务能力
4. 如何通过配置化能力支撑不同行业、不同项目的快速交付

### 三、OMES 系统核心能力

#### 1. 多场景（Scene）建模与管理能力

* 基于“场景（Scene）”的设备组织模型，支持车间、产线、项目等多层级结构；
* 场景与设备解耦，支持设备跨场景复用
* 根据场景维度自动聚合运行、在线、报警等指标数据
* 支持与第三方 SCADA 工艺画面无缝集成，实现“监控画面 + 业务数据”协同展示。

#### 2. 高度配置化的设备管理体系

1. **设备状态与运行态管理**：统一抽象设备运行态 / 在线态 / 报警态；实时监控与历史追溯；支持趋势分析、状态统计与可视化报表
2. **设备属性与数据采集模型**：点位级动态属性配置；实时采集与历史持久化；属性级趋势分析、对比分析与异常识别
3. **设备 GIS 空间化管理**：地理位置建模；地图可视化展示设备分布、状态与告警

#### 3. 统一权限与身份管理体系

* 基于 RBAC（Role-Based Access Control）的权限模型
* 支持管理端与移动端权限隔离
* 场景级、设备级精细化授权，满足多角色协同需求

#### 4. 完整的报警与消息通知机制

* 多级报警规则与事件模型
* 支持系统通知、设备报警、异常事件推送
* 移动端实时接收，提升运维响应效率

### 四、多终端产品形态

| 终端 | 说明 | 典型场景 |
|------|------|----------|
| **OMES-ADMIN 管理平台**（Web） | 自适应工业大屏，支持监控中心部署；全量设备与场景管理；数据分析、配置管理与权限控制统一入口 | 监控中心、运维指挥、系统配置 |
| **OMES 智设备**（微信小程序 / APP） | 设备与场景实时状态查看；报警与系统消息即时触达 | 现场运维、移动办公、移动巡检 |

### 六、总结

OMES 并非传统意义上的“设备云平台”，而是一套面向工业项目交付场景的设备运营管理底座。它以产品化能力降低实施成本，以本地化部署保障数据安全，以高度配置化支撑复杂工业场景，为中小型工业项目提供一条**可规模化复制、可持续演进**的信息化路径。
